**Title:** Learning about the Internal Architecture and Rendering Process of Web Browsers

**Content:**
**[Theoretical Foundation]**
At The Odin Project, we learned about browser compatibility and the operating principles of web browsers.
We gained an overview of the critical rendering path, from URL input to DNS resolution, HTTP request, and response data parsing to the construction of the DOM tree.

**[Insights & Challenges]**
By learning about the behavior and historical background of rendering engines (browser wars and the standardization process), we reaffirmed the current dominance of Chrome (the Blink engine) and the importance of cross-browser compatibility.

The internal structure is extremely complex, and while I currently only have a high-level understanding, I believe a deeper understanding will provide essential foundational skills for optimizing rendering performance and troubleshooting complex bugs.

**【Next Steps】**
This time, I focused on understanding the big picture, but going forward, I'll continue my CS (Computer Science) fundamentals while studying the browser rendering pipeline in more detail and becoming more performance-conscious when writing code.




**タイトル:** フレームワークの有用性と、その裏側にあるCSS/HTML基礎の重要性に関する考察

**内容:**
**【Frameworks vs Fundamentals】**
The Odin Projectにて、CSSフレームワークとプリプロセッサの役割について学習しました。
当初は「便利なフレームワークがあるなら、生のHTML/CSSを深く学ぶコストは不要ではないか」という疑問を持ちました。しかし、ドキュメントや技術記事を読み込む中で、フレームワークはあくまで基礎の上に成り立つ「抽象化レイヤー」に過ぎないと痛感しました。

**【Analogy to Python Experience】**
この構造は、現在の業務でPythonを用いた自動化ツールを開発している経験と完全にリンクしました。
便利なライブラリ（PandasやSelenium等）を活用する場合でも、言語仕様そのものへの理解が浅いと、予期せぬ挙動のデバッグや応用的な実装ができません。Web開発においても同様に、BootstrapやTailwindを使う背後で「CSSがどう解釈されているか」を知らなければ、真のエンジニアリングはできないと再認識しました。

**【Learning Strategy】**
膨大なCSSプロパティを「暗記（Rote Memorization）」しようとして閉塞感を感じていましたが、重要なのは構文の丸暗記ではなく、**「レンダリングの仕組みやレイアウトモデルを概念的に把握すること（Mental Model）」**だと気づきました。辞書を引くようにドキュメントを参照できる状態を目指し、基礎学習の質を変えていきます。


**Title:** Considerations on the Usefulness of Frameworks and the Importance of CSS/HTML Fundamentals

**Content:**
**[Frameworks vs. Fundamentals]**
At The Odin Project, I learned about the role of CSS frameworks and preprocessors.
At first, I wondered, "If there are useful frameworks, isn't there any need to go to the trouble of learning raw HTML/CSS in depth?" However, as I read the documentation and technical articles, I realized that frameworks are merely an "abstraction layer" built on the foundations.

**[Analogy to Python Experience]**
This structure perfectly resonated with my current experience developing automation tools using Python.
Even when using convenient libraries (such as Pandas or Selenium), a shallow understanding of the language specifications makes it difficult to debug unexpected behavior or implement practical applications. Similarly, in web development, I realized that true engineering is impossible without understanding how CSS is interpreted behind the scenes when using Bootstrap or Tailwind.

**[Learning Strategy]**
I felt stuck trying to memorize the vast number of CSS properties, but I realized that the key is not rote memorization of syntax, but rather a conceptual understanding of the rendering mechanism and layout model (mental model). I aim to be able to refer to documentation as easily as looking up a dictionary, and I'm changing the quality of my basic learning.
